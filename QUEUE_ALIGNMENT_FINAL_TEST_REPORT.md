## ⚠️ Известные ограничения

1. **Услуги без врача**: Обрабатываются через обычный cart endpoint (не через batch)
   - **Причина**: Batch endpoint требует `specialist_id` (user_id)
   - **Влияние**: Минимальное, так как большинство услуг имеют врача
   - **Статус**: Приемлемо для текущей версии

2. **Конвертация doctor_id → user_id**: Выполняется на backend
   - **Причина**: Frontend передает `doctor_id`, но batch endpoint требует `user_id`
   - **Решение**: Backend автоматически конвертирует
   - **Статус**: ✅ Работает корректно
