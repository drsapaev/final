# 📊 Отчеты качества кода

Эта директория содержит отчеты о качестве кода, сгенерированные автоматически.

## 📄 Файлы отчетов

- `ALL_ERRORS_REPORT.md` - **Единый отчет со всеми ошибками линтинга**
- `ruff_app.txt` - Ошибки Ruff для директории `app/`
- `ruff_tests.txt` - Ошибки Ruff для директории `tests/`
- `flake8.txt` - Критические ошибки Flake8
- `mypy.txt` - Ошибки типов MyPy
- `pylint.txt` - Ошибки Pylint

## 🔧 Как использовать

### 1. Сбор всех ошибок локально

```bash
cd backend
python scripts/collect_linting_errors.py
```

Этот скрипт:
- Запустит все линтеры
- Соберет все ошибки в единый отчет
- Сохранит отчет в `quality-reports/ALL_ERRORS_REPORT.md`

### 2. Просмотр отчета

Откройте файл `backend/quality-reports/ALL_ERRORS_REPORT.md` - там будут все ошибки с указанием файлов и строк.

### 3. Исправление ошибок

1. Откройте `ALL_ERRORS_REPORT.md`
2. Исправьте все ошибки по порядку
3. После исправления запустите скрипт снова для проверки
4. Когда все ошибки исправлены - закоммитьте изменения

### 4. Автоматическая генерация в CI

В GitHub Actions отчеты генерируются автоматически и доступны как артефакты.

## 📋 Процесс работы

1. **Сбор ошибок** → Запустить скрипт или дождаться CI
2. **Просмотр отчета** → Открыть `ALL_ERRORS_REPORT.md`
3. **Исправление** → Исправить все ошибки из отчета
4. **Проверка** → Запустить скрипт снова
5. **Коммит** → Закоммить исправления

## ⚠️ Важно

- **НЕ коммитьте** по 2-3 ошибки за раз
- **Соберите ВСЕ ошибки** в один отчет
- **Исправьте ВСЕ ошибки** за один раз
- **Проверьте** перед коммитом

## 🎯 Цель

Исправить все ошибки качества кода за один раз, а не по частям.
