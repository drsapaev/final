## 2024-03-14 - Add accessible clearable prop to MacOSInput
**Learning:** Adding a small, accessible clear button to standard inputs drastically improves the data entry experience by making it effortless to reset erroneous inputs without repetitive backspacing. The clear action itself must be treated as an interactive element (`<button>`) with an `aria-label` so screen reader users are aware of its presence and purpose. Furthermore, we must ensure correct sizing so the icon doesn't overlap text, and keyboard interactivity is properly handled with focus states.
**Action:** Always make sure icon-only utility buttons inside form components use a real `<button>` element with `type="button"`, have a descriptive `aria-label`, and show focus indicators (like `box-shadow`) to maintain complete keyboard and screen reader accessibility. Ensure `padding-right` dynamically accounts for the clear button to prevent text overlap.
