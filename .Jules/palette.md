## 2026-03-23 - [MacOSTable Accessibility: Table Header Keyboard Support]
**Learning:** When making table headers (<th>) interactive (e.g., for sorting), it's crucial to retain the native `role="columnheader"` rather than overriding it with `role="button"`. Screen readers rely on `columnheader` to understand the table structure. Instead, make the element focusable with `tabIndex={0}`, handle `onKeyDown` for Space/Enter keys, and use the `aria-sort` attribute (with values like 'ascending', 'descending', or 'none') to communicate the sorting state to assistive technologies.
**Action:** When implementing sortable data tables, always add `tabIndex={0}`, an `onKeyDown` handler for activation, and dynamically update the `aria-sort` attribute on the `<th>` element without changing its native role.
