## 2024-03-01 - Added missing `aria-label` attributes to Sidebar toggles
**Learning:** Reusable navigation elements like Apple's Human Interface Sidebar have multiple nested or dynamically-rendered `<button>` tags that rely entirely on generic icons, which leads to silent failures for screen readers. Dynamic ARIA toggling (e.g. Expand vs Collapse) must evaluate based on current UI state.
**Action:** When creating or modifying navigational sidebars, specifically in components like `Sidebar.jsx`, verify that dynamic visual toggles update their text equivalents accordingly (`aria-label={isCollapsed ? "Expand" : "Collapse"}`) and that all dynamically rendered menu items provide full text representations of their labels to the `aria-label` property (`aria-label={item.label}`).
