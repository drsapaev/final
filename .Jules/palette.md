## 2024-05-20 - Fix clearable input synchronization with React state
**Learning:** When creating a custom `clearable` button inside an input component, directly mutating `ref.current.value = ''` will bypass React's synthetic event system. This leads to form state (e.g., `useState`) not reflecting the cleared value, even if the visual input is empty. To correctly clear an input and notify React, you must set the value tracker, dispatch a native `input` event, and then dispatch a native `change` event.
**Action:** When implementing custom clear buttons or modifying input values programmatically, always manually dispatch `input` and `change` events and update the `_valueTracker` so React state updates accordingly. Additionally, when using `forwardRef`, an internal ref must be used combined with `useImperativeHandle(ref, () => internalRef.current)` to support both callback refs and object refs.
