# 🏗️ Архитектурные Правила Проекта

> **ВАЖНО:** Этот документ содержит ОБЯЗАТЕЛЬНЫЕ правила для всех разработчиков и AI-агентов.

---

## 1. SSOT (Single Source of Truth)

### Принцип: Backend — единственный источник истины

**Frontend НИКОГДА НЕ:**

| Нарушение | Описание | Пример нарушения |
|-----------|----------|------------------|
| **Угадывает** | Выводит данные, которые должен предоставить backend | `if (name.includes('кардио')) dept = 'cardio'` |
| **Агрегирует** | Объединяет/дедуплицирует domain объекты | `mergedByPatientKey`, `appointmentsMap` |
| **Компенсирует** | Исправляет несоответствия на клиенте | `fallback: a || b || c || 'unknown'` |
| **Содержит domain-логику** | Реализует бизнес-правила | Правила membership, приоритеты |

**Frontend МОЖЕТ:**

| Тип | Описание |
|-----|----------|
| **Presentation logic** | Форматирование, сортировка, визуальная группировка |
| **Временные адаптеры** | С пометкой `// ⚠️ TEMPORARY ADAPTER` и ссылкой на тикет |

---

## 2. API Contract

### Backend ОБЯЗАН предоставлять:

| Поле | Описание |
|------|----------|
| `department_key` | Канонический ключ отделения |
| `queue_tag` | Тег очереди |
| `specialty` | Специальность врача |
| `queue_numbers` | Массив номеров в очередях |
| `queue_time` | Время регистрации в очереди |
| `source` | Источник записи (`online`, `desk`, `morning_assignment`) |

### Frontend НЕ ДОЛЖЕН:
- Выводить эти поля из других данных
- Использовать fallback цепочки для определения значений
- Создавать маппинг-словари для преобразования

---

## 3. Правила для AI-агентов

### При написании кода:

1. **Проверяй** — существует ли нужное поле в API
2. **Спрашивай** — если поля нет, спроси пользователя
3. **Не угадывай** — никогда не выводи данные из других полей
4. **Помечай адаптеры** — все временные решения с `// ⚠️ TEMPORARY ADAPTER`

### При code review:

- [ ] Нет `if (field.includes('...'))` для domain логики
- [ ] Нет `Map()` для дедупликации бизнес-сущностей
- [ ] Нет fallback цепочек для domain полей
- [ ] Нет mapping-словарей (`specialtyToX`, `codeToY`)
- [ ] Все адаптеры помечены

### При обнаружении нарушения:

1. **Остановись**
2. **Сообщи пользователю**
3. **Предложи SSOT решение**
4. **Создай тикет для backend** (если данные отсутствуют)
