## 1️⃣ СПИСОК РОЛЕЙ

### Основные роли (из БД):
- `Admin` - Полный доступ ко всем ресурсам
- `Registrar` - Регистрация пациентов, создание визитов
- `Doctor` - Чтение пациентов, работа с EMR, визитами
- `Lab` - Работа с лабораторными результатами
- `Cashier` - Работа с платежами
- `Patient` - Доступ только к собственным данным

### Специализированные роли врачей:
- `cardio` - Кардиолог (права как у Doctor)
- `derma` - Дерматолог (права как у Doctor)
- `dentist` - Стоматолог (права как у Doctor)

### Дополнительные роли:
- `Nurse` - Медсестра
- `Receptionist` - Регистратор
- `Manager` - Менеджер
- `SuperAdmin` - Супер-администратор (обходит все проверки через `is_superuser=True`)

## 5️⃣ АУДИТ-ЛОГИРОВАНИЕ 403

### Автоматическое логирование

Все попытки несанкционированного доступа (403) автоматически логируются в `user_audit_logs`:

```python
UserAuditLog(
    user_id=current_user.id,
    action="ACCESS_DENIED",
    resource_type="patients",  # Извлекается из пути
    resource_id=123,  # Если есть в пути
    old_data=None,
    new_data={
        "required_roles": ["Admin", "Registrar"],
        "user_role": "Patient"
    },
    description="403 Forbidden: POST /api/v1/patients/ - требуется роль: Admin, Registrar, текущая роль: Patient",
    request_id=request.state.request_id,
    ip_address=request.client.host,
)
```

- ✅ Все 403 логируются автоматически через `require_roles()`
- ✅ Запись включает: user_id, action, resource_type, resource_id, required_roles, user_role, request_id, ip_address
