# `aif-commit` Context

Read `../common/SKILL.md` first.

## Project-Specific Notes

- Use conventional commit scopes that match this repo: `backend`, `frontend`, `docs`, `ci`, `ai-factory`, `queue`, `billing`, `emr`, `auth`, `admin`, `settings`, `tests`.
- Changes in `.ai-factory/*` or skill-context should usually use `ai-factory` scope.
- Split commits when AI Factory context work and app work are unrelated.
- Mention port, migration, or runtime changes in the body when they matter to the change.
