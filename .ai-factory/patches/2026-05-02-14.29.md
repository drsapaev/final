# Restore local agent gate for GPT-5.5-era prompts

**Date:** 2026-05-02 14:29
**Files:** `ai/langgraph/scripts/agent_gate.py`, `ai/langgraph/README.md`
**Severity:** high

## Problem

Risky execution tasks were blocked because `AGENTS.md` required
`ai/langgraph/scripts/agent_gate.py`, but the script was absent from the
current worktree. The older gate also appeared to depend on model-specific
prompt behavior that is not appropriate for current GPT-5.5 usage.

## Root Cause

The repository retained the operating contract and evidence history for
`agent_gate`, but not the executable gate itself. That left agents with a hard
stop before risky fixes and no local way to produce first-touch files, stop
conditions, or known-root-cause override prompts.

## Solution

Restored a deterministic local gate at `ai/langgraph/scripts/agent_gate.py`.
The gate accepts `--model`, defaults to `gpt-5.5`, normalizes `ChatGPT 5.5`
aliases, and avoids remote LLM calls. It derives first-touch files from
explicit paths and repo-specific ownership heuristics, supports
`--known-root-cause`, and emits the `Ready-to-send execution prompt` contract
expected by `AGENTS.md`.

## Prevention

- Keep repo-safety tools executable when instructions require them.
- Prefer deterministic local gates for safety boundaries; use model names as
  downstream metadata, not as fragile prompt-runner dependencies.
- Add smoke checks for `agent_gate.py` after changing dev-brain instructions.
