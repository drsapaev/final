# Patch: Frontend prop-types sweep and ThemeContext client import

## Root Cause
- `prop-types` warnings had accumulated across many runtime React components outside `emr-v2`, mostly in shared UI helpers and page-level components.
- `ThemeContext` dynamically imported `frontend/src/api/client.js`, which caused a Vite chunking warning even though the API client is a shared singleton and can be imported statically.

## Fix
- Added a codemod-driven prop-types sweep across warning-bearing frontend runtime files outside `emr-v2`.
- Fixed the remaining test helper warning in `AIAssistant.test.jsx`.
- Replaced the dynamic `client.js` import in `ThemeContext` with a static import of the shared API client.

## Prevention
- Keep sweeping `prop-types` whenever touching shared React helpers or page-level components outside `emr-v2`.
- Use static imports for shared singleton clients to avoid Vite chunking warnings and preserve consistent module initialization.
