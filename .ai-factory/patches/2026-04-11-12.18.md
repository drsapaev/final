# Agent gate known-root-cause escape hatch

**Date:** 2026-04-11 12:18
**Files:** `ai/langgraph/scripts/agent_gate.py`, `ai/langgraph/README.md`, `AGENTS.md`
**Severity:** medium

## Problem

`agent_gate` could repeatedly route a confirmed narrow fix to the wrong owner. In the registrar payment/status case, the gate preferred QR queue files and blocked the known source file `backend/app/api/v1/endpoints/registrar_wizard.py`.

## Root Cause

The gate only validated handoff shape, not whether the handoff included a human-confirmed root-cause file. A valid-looking handoff could therefore be operationally wrong and create a dead-end.

## Solution

Added `--known-root-cause` support. The gate now runs the normal handoff first, retries once with the approved root-cause hint if first-touch files miss it, and emits a narrow override prompt with evidence flags if the retry still misses it.

## Prevention

- For confirmed source fixes, pass the known root-cause file to the gate.
- Treat `Result: gate misroute` as a narrow override, not permission for broad edits.
- Record `gate_misroute`, `override_used`, and `known_root_cause_file` in evidence.
