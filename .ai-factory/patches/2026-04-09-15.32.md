# Public-route websocket gating and silent legacy stubs

**Date:** 2026-04-09 15:32
**Files:** `frontend/src/contexts/NotificationWebSocketContext.jsx`, `frontend/src/components/common/EditPatientModal.jsx`, `frontend/src/hooks/useEMRAutosave.js`
**Severity:** medium

## Problem

The browser console still showed noisy startup behavior:
- notification websocket attempted to connect on `/login`, then immediately closed during development mount/unmount cycles
- a legacy `EditPatientModal` stub logged a deprecation warning on every render
- EMR autosave kept re-scheduling after `accessDenied`, producing repeated "paused" messages

## Root Cause

The notification websocket provider was mounted globally and connected even on public routes that do not need realtime notifications. In React dev/StrictMode, that led to a connect/close cycle before the socket finished establishing.

The legacy modal was implemented as a stub but still emitted a runtime warning inside the render path, so the warning repeated whenever the component mounted.

The autosave hook treated repeated access-denied failures as a recoverable retry state because `canSave()` did not consider the error cap. That allowed the effect to keep re-scheduling autosave even after it had already been paused.

## Solution

Notification websocket startup now checks the current route and skips public pages entirely. On `/login` and other public screens the provider does not connect, so the browser no longer produces the "closed before connection is established" noise from an unnecessary socket lifecycle.

The legacy modal was turned into a silent noop stub with no runtime warning. It still renders a minimal fallback if opened, but it no longer logs every mount.

Autosave now treats the maximum error count as a hard pause state in `canSave()`, preventing repeated timer rescheduling and duplicate "paused" logs after access denial.

## Prevention

- Gate realtime providers by route auth state, not just by token presence.
- Keep deprecated compatibility stubs silent unless there is a one-time migration need.
- If a retry loop has a terminal error state, include that state in the preflight guard so the loop cannot re-arm itself.
