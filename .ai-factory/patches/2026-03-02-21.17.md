# Patch: Remove Legacy SQLite IDE Override

Date: 2026-03-02 21:17

## Problem

The project is documented and configured as Postgres-first, but local IDE debug launches still forced:

```json
"DATABASE_URL": "sqlite:///./clinic.db"
```

This created a split-brain dev setup:
- `backend/.env` pointed to Postgres
- VS Code backend launch configs forced SQLite
- frontend auth logs could therefore come from a SQLite-backed backend even though the repo had already migrated to Postgres

## Root Cause

Stale `.vscode/launch.json` entries overrode `DATABASE_URL` for both backend debug profiles:
- `🚀 FastAPI Backend`
- `🔧 Debug Backend API`

That override won over the intended `backend/.env` Postgres configuration.

## Fix Applied

Updated `.vscode/launch.json`:
- removed hardcoded `DATABASE_URL=sqlite:///./clinic.db`
- added `envFile: ${workspaceFolder}/backend/.env` to both backend launch configs
- kept `PYTHONPATH` explicit

## Verification

- Confirmed `backend/.env` uses `postgresql+psycopg://clinic:clinicpwd@localhost:5432/clinicdb`
- Confirmed `.vscode/launch.json` no longer injects SQLite
- Confirmed local Postgres is reachable and currently contains `1` user row

## Notes

- Current local Postgres does **not** contain `registrar@example.com`; that account exists in legacy SQLite-era scripts/reports, not in the current Postgres data.
- After relaunching the backend from the IDE, auth behavior will now reflect Postgres data, not `clinic.db`.
