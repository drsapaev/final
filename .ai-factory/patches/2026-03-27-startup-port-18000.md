# Patch: backend startup defaults to port 18000

**Date:** 2026-03-27

## Context

The frontend dev proxy and local QA workflow expect the backend to be reachable on `http://localhost:18000`, but several launch scripts still defaulted to `8000`.

## Changes

- Updated `backend/run_server.py` to default to `BACKEND_PORT=18000`
- Updated `backend/start_server.py` to default to `BACKEND_PORT=18000`
- Updated `backend/run_server_auto.py` to default health checks and uvicorn startup to `18000`
- Updated `backend/start_server_verbose.py` to use `BACKEND_PORT` with a default of `18000`
- Updated Windows wrappers:
  - `backend/start_server.bat`
  - `backend/start_server_fixed.bat`
  - `backend/start_server_auto.bat`
  - `backend/restart_server.ps1`
  to set `BACKEND_PORT=18000`

## Verification

- `python -m py_compile` passed for the updated Python launch scripts
- Manual sanity-check confirmed the main launch files now reference `18000`
- Live startup proof: `backend/run_server.py` booted the backend on `127.0.0.1:18000`, `/api/v1/health` returned `200`, and the process was stopped after verification

## Notes

This change aligns startup behavior with the existing frontend proxy and current local QA environment.
