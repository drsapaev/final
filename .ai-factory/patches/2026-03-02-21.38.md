# Patch: Bootstrap Missing User Settings and Remove Security UI Console Noise

Date: 2026-03-02 21:38

## Problem

After switching local development to Postgres and migrating only the `users`
table, several frontend console errors remained:

- `PrintDialog` prop-type warning because `documentType=""` was passed while the
  dialog stayed mounted
- `GET /notifications/settings/:userId` returned `404` because migrated users
  had no `user_profiles` / `user_notification_settings`
- `GET /2fa/security-logs` and `GET /2fa/recovery-methods` returned `500`
  because the endpoints existed but `TwoFactorService` did not implement the
  methods they called

## Root Cause

- Legacy user migration moved only `users`, not related self-service records
  (`user_profiles`, `user_preferences`, `user_notification_settings`)
- Notifications API assumed those rows already existed
- 2FA SMS/email router exposed endpoints backed by missing service methods
- Frontend notification settings screen also incorrectly stored the whole
  Axios response instead of `response.data`

## Fix Applied

### Backend

- Added `UserManagementService.ensure_user_support_records(...)`
  in `backend/app/services/user_management_service.py`
- Wired notifications endpoints to auto-bootstrap missing profile/preferences/
  notification settings on first access
- Added `TwoFactorService.get_security_logs(...)`
  and `TwoFactorService.get_recovery_methods(...)`
  in `backend/app/services/two_factor_service.py`

### Frontend

- Fixed `RegistrarPanel` print dialog state to keep a valid
  `documentType='ticket'` even when closed
- Fixed `NotificationPreferences` to use `response.data`
- Fixed local `Toggle` component to actually call `onChange`
  and made it button-based to avoid click-handler a11y issues

## Data Backfill Performed

Executed a one-time bootstrap for all current Postgres users:

- `user_profiles`: `13`
- `user_preferences`: `13`
- `user_notification_settings`: `13`

Verified user `20` (`registrar@example.com`) now has notification settings.

## Verification

- `pytest backend/tests/unit/test_user_bootstrap_and_two_factor_service.py backend/tests/unit/test_migrate_users_to_postgres.py -q`
  -> `4 passed`
- `npx eslint src/components/settings/NotificationPreferences.jsx src/pages/RegistrarPanel.jsx`
  -> no errors, only one pre-existing warning in `RegistrarPanel.jsx`

## Prevention

- Legacy data migrations must include required dependent user records, not only
  `users`
- New API endpoints should not be added until their service methods exist
- Settings screens should normalize Axios responses to payload data immediately
