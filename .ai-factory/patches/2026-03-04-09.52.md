# Header Theme Menu Clicks Were Stopped In Capture Phase

**Date:** 2026-03-04 09:52
**Files:** frontend/src/components/layout/HeaderNew.jsx, frontend/src/components/layout/__tests__/HeaderNew.test.jsx
**Severity:** medium

## Problem

Тема из списка в хедере визуально выбиралась, но фактически не менялась.
Меню открывалось, однако клик по пунктам вроде `Темная` или `Градиентная палитра`
не обновлял `data-theme` / `data-color-scheme`.

## Root Cause

Корневой контейнер portal-меню использовал `onClickCapture` с
`stopPropagation()`. Из-за этого событие клика обрывалось ещё на capture phase
родительского контейнера и не доходило до `onClick` кнопки конкретной темы.

Это был не баг theme-state и не проблема persistence, а именно ошибка в
обработке событий на уровне UI-контейнера dropdown/portal.

## Solution

- Убран `onClickCapture` с корня theme menu в `HeaderNew.jsx`
- Сохранена безопасная проверка outside-click через `composedPath()`
- Добавлен regression test, который рендерит `HeaderNew`, открывает portal menu
  и подтверждает, что выбор `Темная` реально переключает DOM в `dark`

Дополнительно runtime был проверен headless browser-сценарием с mock JWT и mock
ответами `/auth/me` и `/users/me/preferences`.

## Prevention

- Не использовать `stopPropagation()` на capture phase у корня dropdown/portal,
  если внутри есть кликабельные элементы
- Для outside-click логики проверять `composedPath()`, а не полагаться только на
  `event.target.closest()`
- Для dropdown/portal компонентов всегда иметь regression test на реальный клик
  по menu item, а не только на состояние provider
