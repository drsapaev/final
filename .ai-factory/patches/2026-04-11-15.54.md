# Doctor Integration Payment-State Separation

## Summary
Doctor integration endpoint no longer infers payment completion from visit/appointment completion or writes `appointment.visit_type="paid"` as a payment marker.

## Changed
- Updated `backend/app/api/v1/endpoints/doctor_integration.py`.
- `complete_patient_visit()` now preserves payment state only from existing `Payment` rows.
- Removed automatic paid-payment creation from visit/appointment completion branches.
- Removed `appointment.visit_type="paid"` writes from the appointment completion branch.
- Queue-completion path still preserves `discount_mode="paid"` when a real payment already exists.
