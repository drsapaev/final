# Landing Strategy

This skill converts the brief into a persuasive page strategy. It should produce a section order and message hierarchy that later copy and build skills can implement without re-deciding fundamentals.

## Workflow

1. Read:
   - `briefs/landing_brief.json`
   - `briefs/brand_constraints.json`
2. Define:
   - core promise
   - target segment
   - primary and secondary objections
   - trust strategy
   - CTA ladder
3. Choose a section sequence that serves the CTA.
4. Update `plans/pipeline_plan.md` with section intent and validation priorities.

## Required Outputs

- one-sentence positioning
- hero strategy
- section-by-section goal
- objection handling plan
- trust and proof placement
- CTA map

## Guardrails

- strategy must match the declared audience, not a generic SaaS landing
- each section needs one job only
- if multiple audiences exist, pick one primary audience and demote the rest
