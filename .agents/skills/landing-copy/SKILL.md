# Landing Copy

This skill owns the words on the page. It should write to `content/landing-copy.md` and support `content/seo-meta.json` without changing the underlying strategy.

## Workflow

1. Read:
   - `briefs/landing_brief.json`
   - `briefs/brand_constraints.json`
   - `plans/pipeline_plan.md`
2. Draft copy for:
   - hero
   - supporting sections
   - proof blocks
   - FAQ or objection handling
   - CTA variants
3. Keep sentence rhythm and specificity aligned with the declared audience.
4. Flag any claim that lacks proof.

## Guardrails

- write for one primary audience
- avoid vague hype and empty adjectives
- if proof is missing, lower the claim instead of inventing evidence
