# Pilot Start Checklist

Use this on the first live pilot day before opening the clinic to real users.

## Before Opening
- Confirm the clinic contour is the named real contour.
- Confirm the host machine is the intended clinic host.
- Confirm the database is the intended live clinic database.
- Confirm the official clinic URL is the confirmed LAN or host URL, not `localhost`.
- Confirm the latest backup completed successfully.
- Confirm the last update rehearsal passed.
- Confirm the last restore rehearsal passed.
- Confirm the pilot user list is limited to the approved group.

## Go / No-Go
- If any backup, restore, or update check is missing, do not open the pilot.
- If setup reappears on an initialized instance, do not open the pilot.
- If the live URL or origin is unclear, do not open the pilot.
- If the pilot scope has expanded, pause and re-approve the contour.

## First Hour
- Watch for login failures.
- Watch for queue, registrar, and cashier smoke problems.
- Record any incident note immediately.
- Treat any red smoke as a stop condition.

## End Of Day
- Save the day’s evidence.
- Take a backup.
- Record any open issues.

## Acceptance
- The pilot starts only when the contour, backups, and go/no-go checks are all green.
