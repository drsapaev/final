## 🎯 Обзор системы

EMR V2 — это система электронной медицинской карты с интеллектуальными подсказками, основанными на **реальной практике врача**.

### Ключевые принципы

| Принцип | Описание |
|---------|----------|
| 📜 **Врач — источник истины** | Система учится на подписанных EMR врача |
| 🚫 **Без AI-галлюцинаций** | Подсказки только из реальных данных |
| ⚡ **1 клик = результат** | Минимум действий для частых операций |
| ↩️ **Undo/Redo** | Любое действие можно отменить |

## 📝 Жалобы и Анамнез

### Поле «Жалобы»

**Источники подсказок:**
- 📜 **История врача** — фразы из прошлых EMR этого врача

**Как это работает:**
1. Начните вводить текст (минимум 2 символа)
2. Появятся подсказки из вашей истории
3. Кликните на подсказку для вставки

### Поле «Анамнез заболевания»

Аналогично полю жалоб — подсказки из истории врача.

## 🏥 Диагноз (ICD-10)

### Выбор диагноза

1. Нажмите кнопку 🧠 **AI** или начните вводить
2. AI предложит коды МКБ-10 на основе жалоб
3. Выберите код из списка

### Формат отображения

```
I10 — Эссенциальная [первичная] гипертензия
```

**Важно:** Код ICD-10 используется для:
- Связи с шаблонами лечения (📜 Мой опыт)
- Статистики по диагнозам
- Интеграции с лабораторией

## 📜 Персональная клиническая память

### Что это такое

Система автоматически запоминает ваши назначения и предлагает их при повторном диагнозе.

```
[Подписание EMR с I10] → [Сохранение шаблона] → [Предложение при следующем I10]
```

### Как открыть

1. Выберите диагноз (ICD-10)
2. Кнопка **📜 Мой опыт** появится в секции лечения
3. Нажмите — откроется панель с вашими шаблонами

### Элементы шаблона

```
┌──────────────────────────────────────────────────────┐
│ 📌 ✏️                                                 │
│ Эналаприл 10 мг 2 раза в день...                     │
│ 🔁 часто                         18.01.2026          │
└──────────────────────────────────────────────────────┘
```

| Элемент | Значение |
|---------|----------|
| 📌 | Закрепить (max 3 per diagnosis) — всегда сверху |
| ✏️ | Редактировать шаблон |
| 🔁 часто | Вы используете это назначение чаще всего |
| 🕒 редко | Редко используемое назначение |
| 🕒 Давно не использовал | >12 месяцев без использования |

### Редактирование шаблона

При нажатии ✏️ открывается модальное окно:

```
┌──────────────────────────────────────────────────────┐
│  ✏️ Редактировать шаблон                              │
│                                                      │
│  ┌────────────────────────────────────────────────┐  │
│  │ Эналаприл 10 мг 2 раза в день                 │  │
│  │ Амлодипин 5 мг...                             │  │
│  └────────────────────────────────────────────────┘  │
│                                                      │
│  [♻️ Заменить старый]    [💾 Сохранить как новый]    │
│                                                      │
│  [Отмена]                                            │
└──────────────────────────────────────────────────────┘
```

**Опции сохранения:**
- **♻️ Заменить старый** — обновить текст существующего шаблона
- **💾 Сохранить как новый** — создать новый шаблон, сохранив старый

## ✍️ Подписание EMR

### Процесс подписания

1. Заполните все обязательные поля
2. Нажмите **✍️ Подписать**
3. Подтвердите действие

### Что происходит при подписании

| Действие | Описание |
|----------|----------|
| 🔒 Блокировка | EMR становится read-only |
| 📜 Обучение | Система сохраняет ваши назначения как шаблоны |
| 📊 Версия | Создаётся новая версия в истории |
| 📝 Аудит | Фиксируется время и автор |

### Внесение изменений после подписания

Используйте кнопку **Amend** (Внести изменения):
1. Укажите причину изменения
2. Внесите правки
3. Подпишите повторно

## 🧠 AI-помощник

### Доступные AI-функции

| Поле | AI-функция | Требования |
|------|------------|------------|
| 🔍 Осмотр | Генерация плана осмотра | Заполнены жалобы |

### Важные правила

| ✅ Делает | ❌ Не делает |
|----------|-------------|
| Предлагает варианты | Автоматически вставляет |
| Показывает источник | Скрывает происхождение |
| Сортирует по релевантности | Заменяет решение врача |

### Иерархия источников

```
1. 📜 Мой опыт (ваши прошлые назначения) — приоритет
2. 📋 Шаблоны по специальности
3. 🧠 AI-генерация (если нет истории)
```

## 🔧 Техническая архитектура

### Стек технологий

| Компонент | Технология |
|-----------|------------|
| Frontend | React + Redux |
| Backend | FastAPI (Python) |
| Database | PostgreSQL |
| AI | DeepSeek API |

### Ключевые файлы

```
frontend/
  src/
    components/
      emr-v2/
        EMRContainerV2.jsx      # Главный контейнер
        sections/
          ComplaintsSection.jsx  # Жалобы
          DiagnosisSection.jsx   # Диагноз
          TreatmentSection.jsx   # Лечение + 📜 Мой опыт
    hooks/
      useDoctorTreatmentTemplates.js  # Hook для шаблонов

backend/
  app/
    api/v1/endpoints/
      doctor_templates.py    # API для шаблонов
    services/
      doctor_templates_service.py   # Бизнес-логика
      emr_v2_service.py      # EMR операции
    models/
      doctor_templates.py    # SQLAlchemy модель
```

### API Endpoints (Doctor Templates)

```http
# Получить шаблоны по диагнозу
GET /api/v1/emr/doctor-templates/treatment?icd10=I10

# Удалить шаблон (soft delete)
DELETE /api/v1/emr/doctor-templates/treatment/{id}

# Закрепить шаблон
POST /api/v1/emr/doctor-templates/treatment/{id}/pin

# Открепить шаблон
DELETE /api/v1/emr/doctor-templates/treatment/{id}/pin

# Редактировать шаблон
PUT /api/v1/emr/doctor-templates/treatment/{id}
    Body: { "treatment_text": "...", "mode": "replace" | "save_as_new" }
```

### Схема базы данных

```sql
CREATE TABLE doctor_treatment_templates (
    id VARCHAR(36) PRIMARY KEY,
    doctor_id VARCHAR(36) NOT NULL,
    icd10_code VARCHAR(10) NOT NULL,
    treatment_text TEXT NOT NULL,
    treatment_hash VARCHAR(64) NOT NULL,
    usage_count INTEGER DEFAULT 1,
    last_used_at DATETIME,
    created_at DATETIME,
    is_deleted BOOLEAN DEFAULT FALSE,
    deleted_at DATETIME,
    is_pinned BOOLEAN DEFAULT FALSE,
    pinned_at DATETIME,
    UNIQUE(doctor_id, treatment_hash)
);
```

## 📊 Метрики и аналитика

### Что отслеживается (backend)

- Количество шаблонов на диагноз
- % используемых шаблонов (usage_count > 1)
- Pinned vs не-pinned
- Stale шаблоны (>12 месяцев)

### Телеметрия (frontend)

```javascript
onTelemetry({ 
    type: 'treatment.my_experience.applied', 
    payload: { icd10: 'I10' } 
});
```

## 🛡️ Безопасность и соответствие

### Принципы

| Принцип | Реализация |
|---------|------------|
| Персональность | Шаблоны привязаны к doctor_id |
| Аудит | Все действия логируются |
| Soft delete | Данные не удаляются физически |
| Версионирование | История всех изменений EMR |

### Ограничения

- ❌ Врач НЕ видит шаблоны других врачей
- ❌ AI НЕ генерирует «от себя» — только на основе данных
- ❌ Автоподстановка ЗАПРЕЩЕНА — только по клику
