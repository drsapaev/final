# AI Smoke Tests

> Прогнать перед каждым релизом, влияющим на AI функциональность

---

## ✅ Чеклист

| # | Сценарий | Шаги | Ожидание | ✓ |
|---|----------|------|----------|---|
| 1 | **AI подсказки для жалоб** | Ввести жалобу → нажать 💡 | Появляется popover с подсказками | ☐ |
| 2 | **Применение подсказки** | Выбрать подсказку в popover | Текст вставляется в поле | ☐ |
| 3 | **Undo после AI** | После вставки нажать Ctrl+Z | Текст откатывается | ☐ |
| 4 | **ICD-10 подсказки** | В поле диагноза ввести симптом → AI | Предлагаются коды МКБ-10 | ☐ |
| 5 | **Timeout handling** | Отключить сеть → запросить AI | Показывается ошибка, EMR работает | ☐ |
| 6 | **Долгий ответ** | Подождать > 10 сек | Показывается "AI думает..." | ☐ |
| 7 | **EMR без AI** | Заполнить EMR без использования AI | Всё сохраняется корректно | ☐ |
| 8 | **Ghost Mode (если есть)** | Включить Ghost Mode → ввести текст | Появляется ghost suggestion | ☐ |

## 🔴 Критические сценарии (блокируют релиз)

| # | Сценарий | Почему критично |
|---|----------|-----------------|
| 1 | AI НЕ вставляет текст автоматически | Безопасность: врач контролирует |
| 2 | Undo работает после AI | Исправление ошибок |
| 3 | EMR работает без AI | Graceful degradation |

## 📋 Как прогонять

1. Открыть EMR V2 Demo: `/emr-v2-demo`
2. Пройти по каждому пункту
3. Отметить ✓ или записать баг
4. Если критический сценарий провален → **НЕ релизить**
