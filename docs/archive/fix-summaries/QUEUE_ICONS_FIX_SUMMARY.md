# Исправление иконок очереди (желтый круг с часами) на вкладках

## Проблема
Иконки очереди (🟡⏰) на вкладках регистратуры исчезали после перезагрузки бэкенда.

## Причина
1. **Неправильный импорт модели в бэкенде** - `from app.models.queue import QueueEntry` (файл не существует)
2. **Неправильная проверка hasActiveQueue** - проверялся статус `'queued'` вместо наличия `queue_numbers`

## Исправления

### 1. Бэкенд: Правильный импорт QueueEntry
**Файл:** `c:\final\backend\app\api\v1\endpoints\registrar_integration.py`
**Строка:** 1218

```python
# ❌ БЫЛО (файл queue.py не существует):
from app.models.queue import QueueEntry

# ✅ ИСПРАВЛЕНО:
from app.models.queue_old import QueueEntry
```

**Эффект:** Бэкенд теперь правильно получает реальные номера очереди из таблицы `queue_entries`.

---

### 2. Фронтенд: Проверка queue_numbers вместо статуса
**Файл:** `c:\final\frontend\src\pages\RegistrarPanel.jsx`
**Строка:** 2356-2357

```javascript
// ❌ БЫЛО:
hasActiveQueue: deptAppointments.some(a => a.status === 'queued')

// ✅ ИСПРАВЛЕНО:
hasActiveQueue: deptAppointments.some(a => a.queue_numbers && a.queue_numbers.length > 0)
```

**Эффект:** Иконки появляются для всех записей с `queue_numbers`, независимо от статуса.

## Техническая информация

### Как работают иконки

1. **Бэкенд** (`registrar_integration.py:1215-1240`):
   - Запрос к `queue_entries` для получения реального номера
   - Возврат `number` в JSON ответе

2. **Фронтенд** (`RegistrarPanel.jsx:1221-1229`):
   - Создание массива `queue_numbers` с `queue_name` для каждой записи
   - Сохранение в state `appointments`

3. **Статистика** (`RegistrarPanel.jsx:2342-2362`):
   - Вычисление `departmentStats` для каждого отдела
   - `hasActiveQueue` проверяет наличие `queue_numbers`

4. **Компонент вкладок** (`ModernTabs.jsx:136-146`):
   - Получает `departmentStats` как props
   - Отображает иконку часов если `hasActiveQueue === true`
