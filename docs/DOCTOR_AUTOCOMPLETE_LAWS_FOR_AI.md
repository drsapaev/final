Эта система работает с **конфиденциальными медицинскими данными** и должна соблюдать **строгие принципы приватности и безопасности**.

## 🔴 ЗАКОН №1: НИКАКОЙ ГЕНЕРАЦИИ ТЕКСТА

### ❌ ЗАПРЕЩЕНО:
```python
# НЕТ! Это генерация текста!
def generate_suggestion(prompt):
    response = openai.complete(prompt)
    return response.text

# НЕТ! Это AI-генерация!
def improve_text(doctor_text):
    return llm.enhance(doctor_text)
```

### ✅ РАЗРЕШЕНО:
```python
# ДА! Это поиск в истории
def search_suggestions(doctor_id, prefix):
    return db.query(DoctorPhraseHistory).filter(
        DoctorPhraseHistory.doctor_id == doctor_id,
        DoctorPhraseHistory.prefix_3 == prefix[:3]
    ).order_by(
        DoctorPhraseHistory.usage_count.desc()
    ).all()
```

### ПРИНЦИП:
**"Search & Rank, NOT Generate"**
- ✅ Искать существующие фразы врача
- ✅ Ранжировать по частоте
- ❌ НЕ создавать новый текст
- ❌ НЕ использовать AI/LLM для генерации
- ❌ НЕ "улучшать" текст врача

## 🔴 ЗАКОН №2: АБСОЛЮТНАЯ ИЗОЛЯЦИЯ ДАННЫХ

### ПРАВИЛО:
**Каждый врач видит ТОЛЬКО свою историю. Никаких исключений.**

### ❌ ЗАПРЕЩЕНО:
```python
# НЕТ! Утечка данных между врачами!
def get_suggestions(prefix):
    # Ищем у ВСЕХ врачей - НЕДОПУСТИМО!
    return db.query(DoctorPhraseHistory).filter(
        DoctorPhraseHistory.phrase.startswith(prefix)
    ).all()

# НЕТ! "Обучение" на данных других врачей!
def learn_from_all_doctors():
    all_phrases = db.query(DoctorPhraseHistory).all()
    model.train(all_phrases)  # НАРУШЕНИЕ ПРИВАТНОСТИ!
```

### ✅ РАЗРЕШЕНО:
```python
# ДА! Строгая изоляция по doctor_id
def get_suggestions(doctor_id, prefix):
    return db.query(DoctorPhraseHistory).filter(
        DoctorPhraseHistory.doctor_id == doctor_id,  # ← ОБЯЗАТЕЛЬНО!
        DoctorPhraseHistory.prefix_3 == prefix[:3]
    ).all()
```

### ПРОВЕРКА:
Перед любым изменением спросите себя:
> "Может ли врач А увидеть данные врача Б при этом изменении?"

Если ответ "ДА" или "ВОЗМОЖНО" — **НЕ ДЕЛАЙТЕ ЭТОГО!**

## 🔴 ЗАКОН №3: READINESS — ЭТО СВЯТОЕ

### ПРАВИЛО:
**Автодополнение активируется ТОЛЬКО при выполнении критериев. Никаких ручных переключателей до activation!**

### ❌ ЗАПРЕЩЕНО:
```jsx
// НЕТ! Позволяет включить без данных!
<EMRSmartField
  useDoctorHistory={userSettings.enableHistory}  // WRONG!
  doctorId={doctorId}
/>

// НЕТ! Обход readiness check!
if (userSettings.forceEnable || ready) {
  showSuggestions();  // НЕДОПУСТИМО!
}
```

### ✅ РАЗРЕШЕНО:
```jsx
// ДА! Только через readiness
const { ready } = useDoctorPhrases({ doctorId });

<EMRSmartField
  useDoctorHistory={true}  // статично
  doctorId={doctorId}
/>

// Внутри hook:
if (!ready) {
  return [];  // Нет данных — нет подсказок
}
```

### КРИТЕРИИ (НЕ МЕНЯТЬ БЕЗ МЕДИЦИНСКОЙ ЭКСПЕРТИЗЫ!):
```python
MIN_COMPLETED_EMRS = 10       # Минимум завершённых EMR
MIN_STORED_PHRASES = 30       # Минимум уникальных фраз
MIN_REPEATED_PHRASES = 5      # Минимум повторяющихся фраз
```

**Эти числа выбраны не случайно!** Они обеспечивают:
- Статистическую значимость
- Качество подсказок
- Безопасность (нет галлюцинаций на малых данных)

## 🔴 ЗАКОН №4: ТОЛЬКО ИНДЕКСАЦИЯ, НЕ ИЗМЕНЕНИЕ

### ПРАВИЛО:
**Мы индексируем то, что врач написал. Мы НЕ исправляем, НЕ нормализуем, НЕ "улучшаем".**

### ❌ ЗАПРЕЩЕНО:
```python
# НЕТ! Изменение текста врача!
def index_phrase(phrase):
    # "Улучшение" орфографии - НЕДОПУСТИМО!
    corrected = spell_checker.fix(phrase)
    db.add(DoctorPhraseHistory(phrase=corrected))
    
# НЕТ! "Нормализация" медицинских терминов!
def normalize_medical_term(phrase):
    # Замена "ГБ" на "Гипертоническая болезнь" - НЕТ!
    return phrase.replace("ГБ", "Гипертоническая болезнь")
```

### ✅ РАЗРЕШЕНО:
```python
# ДА! Сохраняем как есть
def index_phrase(phrase):
    # Храним ТОЧНО как написал врач
    db.add(DoctorPhraseHistory(phrase=phrase.strip()))
    
# ДА! Минимальная очистка (только пробелы)
clean_phrase = phrase.strip()
```

### ПРИНЦИП:
**Врач - эксперт. Мы не исправляем медицинские записи.**

## 🔴 ЗАКОН №5: ЗАЩИТА ОТ SQL INJECTION И XSS

### ПРАВИЛО:
**Все пользовательские данные ОБЯЗАТЕЛЬНО проходят через ORM или sanitization.**

### ❌ ЗАПРЕЩЕНО:
```python
# НЕТ! SQL Injection!
def search_phrases(doctor_id, prefix):
    query = f"SELECT * FROM doctor_phrase_history WHERE prefix_3 = '{prefix}'"
    return db.execute(query)  # ОПАСНО!
```

### ✅ РАЗРЕШЕНО:
```python
# ДА! Параметризованные запросы через ORM
def search_phrases(doctor_id, prefix):
    return db.query(DoctorPhraseHistory).filter(
        DoctorPhraseHistory.doctor_id == doctor_id,
        DoctorPhraseHistory.prefix_3 == prefix[:3]  # автоматический escape
    ).all()
```

### FRONTEND:
```jsx
// ДА! Все данные экранируются React автоматически
<div>{suggestion.text}</div>

// НЕТ! Bypass React escaping
<div dangerouslySetInnerHTML={{__html: suggestion.text}} />  // ОПАСНО!
```

## 🔴 ЗАКОН №6: НЕ ТРОГАЙТЕ INDEXABLE_FIELDS БЕЗ ПРИЧИНЫ

### ТЕКУЩИЕ ПОЛЯ (установлены сознательно):
```python
INDEXABLE_FIELDS = [
    'complaints',       # Жалобы
    'anamnesis',        # Анамнез
    'examination',      # Объективный осмотр
    'diagnosis',        # Диагноз
    'recommendations'   # Рекомендации
]
```

### ❌ НЕ ДОБАВЛЯЙТЕ:
- `medications` - это НЕ текст, это структурированные данные
- `procedures` - аналогично
- `vital_signs` - это JSON, не текст
- Поля с PII (персональные данные пациента)

### ✅ МОЖНО ДОБАВИТЬ (с обоснованием):
- `treatment` - если это текстовое поле (проверьте модель!)
- Другие текстовые поля EMR (НЕ JSON!)

### ПЕРЕД ДОБАВЛЕНИЕМ СПРОСИТЕ:
1. Это текстовое поле? (не JSON, не числа)
2. Это пишет врач? (не пациент, не система)
3. Это может повторяться? (есть смысл индексировать)
4. Это безопасно? (нет PII, нет паролей)

## 🔴 ЗАКОН №9: НЕ ДОБАВЛЯЙТЕ "УМНЫЕ" ФИЧИ

### ❌ НЕДОПУСТИМЫЕ "УЛУЧШЕНИЯ":
```python
# НЕТ! "Умное" предсказание следующего слова
def predict_next_word(current_text):
    return markov_chain.predict(current_text)

# НЕТ! "Автокоррекция" медтерминов
def autocorrect_medical(text):
    return medical_spellchecker.fix(text)

# НЕТ! "Синонимы" для разнообразия
def suggest_synonyms(phrase):
    return thesaurus.get_synonyms(phrase)
```

### ПОЧЕМУ НЕТ:
1. **Мы не AI assistant** — мы search engine по истории
2. **Врач знает лучше** — не исправляем его записи
3. **Медицина ≠ autocorrect** — "ГБ" может быть правильно
4. **Простота = надёжность** — меньше кода, меньше багов

## 🔴 ЗАКОН №10: ДОКУМЕНТИРУЙТЕ ИЗМЕНЕНИЯ

### ПРАВИЛО:
**Любое изменение в логике поиска/ранжирования ОБЯЗАТЕЛЬНО документируется.**

### ОБЯЗАТЕЛЬНО УКАЖИТЕ:
```python
# ✅ ХОРОШАЯ ДОКУМЕНТАЦИЯ
def rank_suggestions(phrases):
    """
    Ранжирование подсказок по частоте использования.
    
    Алгоритм:
    1. Сортировка по usage_count (DESC)
    2. При равенстве — по last_used (DESC)
    
    КРИТИЧНО: НЕ используем ML/AI для ранжирования!
    Причина: прозрачность, предсказуемость, медбезопасность.
    
    Изменено: 2026-01-06 - добавлена сортировка по last_used
    """
    return sorted(phrases, 
                  key=lambda p: (p.usage_count, p.last_used), 
                  reverse=True)
```

### ❌ ПЛОХАЯ ДОКУМЕНТАЦИЯ:
```python
def rank_suggestions(phrases):
    # Sort phrases
```
